SECTION 05 · ONE-CLICK FILL

一键填写的三个根因三个修

从用户视角："封面没显示、账号没复原、表单没填上"。三个独立的 bug，串成一条"端到端填不进去"的故事。

// BUG #1：封面 URL 错

  • 旧：http://host:5409/uploads/...

    Vite proxy 没转发 /uploads/ 路径，封面 404。

  • 新：/api/materials/file/{path}

    跟 Vite proxy 走 /api 通到 5409 后端，封面正常加载。

// BUG #2：账号不复原

  • handleOneClickFill 不动 publishAccountIds

    原代码只填 platformConfigs，账号选择保持用户当前态。

  • 按 channels 自动勾选

    清空当前选中 → 遍历 record.channels → 找对应平台 → 把所有匹配账号加入集合。

// BUG #3：表单没填上

  • 把 account_configs 当平台嵌套 dict

    实际是单 detail 扁平配置（{title, description, tags...}），字符串被 typeof === 'object' 过滤掉。

  • 中英文 key 不匹配

    platformConfigs 用 douyin 英文 key，channels[].platform 是抖音中文名，写入不存在的 key。

  • 按 channels 逐个平台填 + platformNameToKey

    遍历 channels 拿 platformNameToKey 转换后填 platformConfigs[key] = {...config}。

修复后效果：
点"从历史一键填写" → 选中模板 → 封面图正常显示 + 平台对应账号被自动勾选 + 标题/描述/标签填入表单。一次点击完成"选历史 → 用历史"完整链路。
