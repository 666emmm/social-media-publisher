SECTION 05 · ACCOUNTSIDEBAR

一个组件，两种模式

AccountSidebar 加 mode='edit'|'readonly' prop。编辑模式给 PublishCenter 用（可选可移除）；只读模式给详情页用（只看不操作）。一个组件，两套行为。

MODE · 'edit'

PublishCenter

  • 底部显示"+ 添加账号"按钮
  • 分组计数 = 该组"已勾选"账号数
  • 账号列表只显示 publishAccountIds 中的
  • 显示 StarFilled 自定义配置图标
  • 显示 Close 移除按钮
  • 点击账号触发 select-account（编辑态）
MODE · 'readonly'

PublishHistoryDetail

  • 底部不显示"+ 添加账号"按钮
  • 分组计数 = 该组"全部"账号数
  • 账号列表显示该组所有账号（不论是否被本次发布选中）
  • 不显示 StarFilled / Close 图标
  • 点击账号触发 select-account（详情态：切换当前详情）
  • 列表空时仍显示"暂无账号"占位
AccountSidebar.vue · mode 关键差异
<!-- 同一段模板，v-if 切换 --> <span class="group-count"> {{ mode === 'readonly' ? group.accounts.length : group.accounts.filter(a => publishAccountIds.has(a.id)).length }} </span> <el-icon v-if="hasAccountOverride(account.id) && mode === 'edit'" class="override-icon"><StarFilled /></el-icon> <el-icon v-if="mode === 'edit'" class="account-remove"><Close /></el-icon> <div v-if="mode === 'edit'" class="sidebar-footer"> <div class="add-btn" @click="$emit('open-account-dialog')">+ 添加账号</div> </div>
分支一致性：PublishCenter.vue 显式 :mode="'edit'"，行为零变化（默认值就是 edit，但显式更可读、避免后人误改）。
